# Estratégia de Comunidade

## Objetivo
Construir uma comunidade técnica ativa, acolhedora e orientada a execução.

## Rituais

- Triagem semanal de issues.
- Office hours quinzenal.
- RFC review mensal.

## Programas

- `Contributor Ladder`: trilha de crescimento (contribuidor → reviewer → maintainer).
- `Good First Month`: lote mensal de issues para iniciantes.
- `Community Champion`: reconhecimento trimestral.

## Canais

- GitHub Discussions para arquitetura e dúvidas.
- Discord para suporte rápido.
- Newsletter mensal de releases e roadmap.

## Métricas

- Tempo médio de resposta.
- PRs externas/total de PRs.
- Taxa de retenção de contribuidores (90 dias).
